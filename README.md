# portwarden

에이전트나 개발 서버가 남겨둔 LISTEN 포트를 빠르게 확인하려고 만든 작은 CLI입니다.

AI가 코딩하고 테스트한다고 싸지른 포트들 찾느라 빡쳐서 만들었습니다.

설치 후 `portwarden`만 실행하면 바로 TUI가 열립니다.

## 설치

```bash
npm install -g portwarden
```

전역 설치 없이도 실행할 수 있습니다.

```bash
npx portwarden
```

## 실행

```bash
portwarden
```

로컬에서 개발 중이면 이렇게 실행하면 됩니다.

```bash
cd /Users/laonpeople/Downloads/dev-port-watch
node dev-ports.js
```

## 할 수 있는 것

- 지금 떠 있는 개발 포트 보기
- 포트별 프로세스/경로 확인
- 전체 LISTEN 포트 보기
- 브라우저로 바로 열기
- 정상 종료 / 강제 종료
- 다음 빈 포트로 프로세스 이동
- 브라우저 / 확인 모드 설정 저장

## 키 조작

- `j` / `k` 또는 방향키: 이동
- `enter`: `ALL` 뷰에서 앱 그룹 펼치기 / 접기
- `←` / `→`: 일반 항목 순서 바꾸기, 앱 그룹은 `→` 펼치기 / `←` 접기
- `a`: 기본 보기와 전체 보기 전환
- `o`: 선택한 포트를 브라우저로 열기
- `m`: 선택한 프로세스를 다음 빈 포트로 옮기기
- `p`: 선택한 포트 pin / 해제
- `x`: 선택한 프로세스 종료 요청
- `f`: 선택한 프로세스 강제 종료
- `s`: 설정 화면 열기
- `q`: 종료

기본값은 원클릭 실행입니다. `o`, `m`, `x`, `f` 모두 바로 동작합니다.

설정 화면에서는 기본 브라우저와 `Confirm mode`를 저장할 수 있습니다.

목록 순서와 상단 고정 상태도 설정 파일에 저장되므로, 다음 실행 때 그대로 복원됩니다.

기본 화면은 `핀한 항목 + dev처럼 보이는 포트`를 보여줍니다. `a`를 누르면 전체 LISTEN 포트로 전환되고, 여기서 `p`로 핀한 포트는 다음부터 기본 화면 상단에도 계속 보입니다.

`ALL` 뷰에서는 Electron 앱, 에이전트, language server 같은 보조 리스너를 앱 단위로 묶어서 보여줍니다. 묶인 줄에서 `enter` 또는 `→`로 펼치고, `enter` 또는 `←`로 접을 수 있습니다.

## 동작 방식

- `x`: 정상 종료 요청을 보내고 정리 작업을 할 기회를 줍니다
- `f`: 바로 강제 종료합니다. `x`로 안 내려갈 때만 쓰는 쪽이 안전합니다
- `m`: 같은 명령을 다음 빈 포트로 다시 띄우고, 새 포트가 실제로 열린 뒤 기존 프로세스를 내립니다

선택한 프로세스의 다음 빈 포트는 상세 패널의 `next`에서 항상 확인할 수 있습니다.

## 표시 기준

기본적으로는 `핀한 항목`과 `개발 서버처럼 보이는 리스너`를 보여줍니다.

- 흔한 개발 포트 사용
- `node`, `bun`, `python`, `uvicorn` 같은 런타임 사용
- 홈 디렉터리 아래 프로젝트 폴더에서 실행된 흔적이 있음
- `vite`, `next`, `webpack`, `storybook` 같은 키워드가 명령어에 포함됨

전체 LISTEN 포트를 보려면 `a`를 누르거나 `portwarden --all`로 실행하면 됩니다.

## 참고

한글 두벌식 입력 상태에서도 같은 키 위치로 단축키가 동작합니다. 다만 터미널 TUI는 물리 키 위치가 아니라 최종 입력 문자를 받는 경우가 많아서, 모든 키보드 레이아웃을 자동으로 같은 방식으로 맞추는 건 현실적으로 어렵습니다. 필요한 레이아웃은 별도로 추가하는 방식이 맞습니다.
